# Docs Update

ドキュメントを最新の変更に合わせて正確に更新する。

## 目的

- 変更内容を正しく反映する。
- 読者が迷わないように手順と要点を整理する。
- 既存の書き方や規約に合わせて一貫性を保つ。

## 対象外

- 軽微な表記修正のみで、更新方針の整理が不要な場合。

## 前提確認

- 更新対象（README・CONTRIBUTING・CHANGELOG・その他）を確認する。
- 変更内容と対象読者を確認する。
- 既存の文体・用語・フォーマットを確認する。
- 変更元の情報（実装差分、仕様、Issue・PR）を確認する。
- 不明点は最小限の質問で補う。

## 手順

### 1) 変更内容の把握

- 実装差分や仕様変更を整理する。
- 影響範囲とユーザーへの影響を把握する。
- 対象読者（開発者・利用者・運用者）を意識する。

### 2) 反映方針の決定

- 既存の構成を崩さないように差し込む。
- 既存の規約（表記・見出し・テンプレ）に合わせる。

### 3) 更新

- 追加・変更・削除を最小限の差分で行う。
- 手順はステップ順に整理する。
- 例や注意点が必要なら簡潔に追加する。

### 4) 整合性確認

- 他のドキュメントと矛盾していないか確認する。
- リンク、コマンド、パスが正しいか確認する。
- リンク先やコマンドが実際に有効か確認する。

## 出力フォーマット

```markdown
# ドキュメント更新: [対象]

## 変更内容

- [要約]

## 反映箇所

- [ファイル・セクション]

## 更新方針

- [方針]

## 追加・変更・削除

- 追加: [内容]
- 変更: [内容]
- 削除: [内容]
```

## 返答のガイド

- 変更点と反映先を簡潔に報告する。
- 影響がある手順や設定変更は明確に書く。
- 実行が必要なコマンドがあれば明示する。
- 不明点があれば短く確認する。
